# 2026-04-13 Update — Agent vs Manual Report Gap Analysis & Fixes

## Overview

Comprehensive comparison of the autonomous agent report against the manual team's forensic report for the Apex Global Logistics ransomware incident. Three systemic root causes were identified and fixed.

---

## Root Causes (All Gaps Traced to 3 Issues)

### 1. Stale `pcap_analysis.json` Cache
- Previous session fixed bugs in `pcap_deep_analysis.py` (RDP `rt_cookie` field name, TCP conversation unit parser, internal-IP SMB filter)
- But the cached output from the OLD buggy code was still being used
- **Result:** 0 rows in `pcap_smb`, `pcap_rdp`, `pcap_tcp_conv` DB tables
- **Fix:** Delete `data/output/*/ingest/pcap_analysis.json` and re-run

### 2. Reservoir Sampling at 500K Cap Destroying Rare Evidence
- `zeek_searcher.py` caps each protocol bucket at 500,000 records via reservoir sampling
- 4 protocols hit the cap: connection (500K), dns (500K), ssl (500K), rdp (500K)
- Total matched: 9,291,243 — meaning ~8.3M records were randomly discarded
- **Critical losses:**
  - 19/47 temp.sh DNS queries lost from `zeek_dns`
  - 0/11 temp.sh TLS sessions in `zeek_ssl`
  - 77.90.153.30 March 8 return RDP session missing from `zeek_conn`
- **Fix:** Added "protected records" mechanism — exfil-related records (temp.sh, file.io, transfer.sh IPs + hostnames) and beachhead-external RDP/SSL/Kerberos records are exempt from reservoir sampling

### 3. Alert-Timestamp PCAP Selection Too Restrictive
- `pcap_selector.py` only selected PCAPs overlapping Suricata alert timestamps
- PCAPs containing temp.sh DNS/TLS, SMB file access, and exfil traffic outside alert windows were skipped
- **Fix:** Changed `ingest.py` to process ALL 129 PCAPs (manual team processed all of them too)

## Fixes Applied

### 1. `tools/zeek_searcher.py` — Protected Records (DONE)
- Added `protected` dict parallel to `records` — exempt from reservoir sampling
- Exfil markers: temp.sh, file.io, transfer.sh hostnames in DNS/TLS/HTTP
- Exfil IPs: 51.91.79.17, 65.22.162.9, 65.22.160.9, 144.76.136.153, 144.76.136.154, 95.216.22.32
- Beachhead-external: RDP/SSL/Kerberos records involving 10.128.239.57 with external IPs
- Conn records: only protect beachhead + external on port 3389 (avoid protecting millions of C2 conn records)

### 2. `tools/ingest.py` — Process ALL PCAPs (DONE)
- Changed from `select_pcaps(pcap_index, alert_timestamps)` to `[e["path"] for e in pcap_index]`
- All 129 PCAPs now processed in Phase 4

### 3. `tools/pcap_deep_analysis.py` — Timeout Increases (DONE)
- tshark field extraction: 180s → 600s
- TCP conversation: 300s → 600s

### 4. Dual-LLM Report Synthesis (DONE)
- `agents/synthesizer.py` modified to run GPT-4o + CommonStack (Claude Opus 4.6) in parallel
- Produces two reports: `report.md` (GPT-4o) and `report_commonstack.md` (Claude Opus 4.6)
